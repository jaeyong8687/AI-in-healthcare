**Role**
당신은 UX 리서처이자 질적 데이터 분석 전문가입니다.
당신의 목표는 데이터를 정리하거나 요약하는 것이 아니라, 주어진 사용자 인터뷰 데이터만을 근거로 하나의 가설이 지지되는지, 부분적으로 수정되어야 하는지, 기각되어야 하는지를 판단하는 것입니다.

주의:
- 가설을 사실로 가정하지 마세요.
- 데이터에 없는 추론이나 해결책 제안은 금지합니다.
- “대부분”, “일반적으로” 같은 모호한 표현을 피하세요.
- 반드시 발화 증거에 근거해 판단하세요.
- 결과 해석을 넘어선 솔루션 제안은 하지 마세요.

**INPUT**
(예시)

Problem: 고령의 당뇨환자들이 복잡한 사용자 인터페이스 로 인해 혈압 및 혈당 데이터를 기록하는 데 어려움을 겪고 있을 것이다. 

2. 데이터:

당뇨 진단: 10년

Q: 평소 혈당이나 혈압 관리는 어떻게 하고 계세요?

Q: 기록은 어떻게 하시나요?
예전에는 공책에 썼는데 요즘은 잘 안 써요. 재고 나면 숫자 보고 그냥 머릿속으로 기억하려고 해요.

병원 가기 전에는 좀 써야겠다 생각하죠. 의사가 물어보면 할 말이 있어야 하니까요.
Q: 기록하다가 그만둔 경험이 있나요?
앱으로ㄷ 해보려고 했는데, 뭘 눌러야 하는지 모르겠더라고요. 혈당만 쓰면 될 줄 알았는데 다른 것도 계속 나오니까 그냥 껐어요.
Q: 만약 기록이 아주 간단해도 계속 하실 것 같나요?
글쎄요… 간단하면 좀 더 하긴 하겠죠. 근데 매일 쓰는 게 솔직히 귀찮긴 해요.

재긴 거의 매일 했어요. 기록은… 한 이틀? 나머지는 그냥 재고 말았어요.
Q: 왜 기록을 안 하게 되셨나요?
숫자들이 다 비슷비슷해요. 써도 뭐가 달라지는지 잘 모르겠어요.
Q: 앱 사용 경험은 있으신가요?
딸이 깔아준 게 있었는데, 글씨가 작아서 잘 안 보여요. 괜히 잘못 누를까 봐 무서워서요.
Q: 기록이 잘 되고 있다고 느끼는 상태는 어떤 건가요?
병원 가서 의사가 “잘 관리하고 계시네요” 이렇게 말해주면 그때는 잘했다 싶죠.

Q: UI가 단순해지면 해결될까요?
어느 정도는요. 근데 기록해도 딱히 피드백이 없으면 오래 하긴 힘들 것 같아요.

**TASK — DATA-DRIVEN SENSEMAKING**
아래 순서를 반드시 따르세요.

1. 데이터에서 반복적으로 관찰되는 행동, 인식, 감정을
   해석 없이 그대로 추출하세요.
   (초기 코드 수준, 해석 금지)

2. 위 관찰 항목들을 묶어
   패턴(candidate themes)을 도출하세요.
   이때 상충되거나 예외적인 패턴도 반드시 포함하세요.

3. 각 패턴이
   가설의 어떤 구성요소
   (전제 / 관찰 / 인과 / 반증)에 해당하는지 매핑하세요.

4. 가설을 지지하는 데이터와
   가설을 약화하거나 반박하는 데이터를
   명확히 구분하세요.
   (각각 발화 인용 포함)

5. 위 매핑을 근거로,
   가설의 상태를 다음 중 하나로 판단하세요:
   - 지지됨
   - 부분 지지 (수정 필요)
   - 기각

**OUTPUT FORMAT**

출력은 다음 구조를 따르세요:

1. 관찰된 주요 행동/인식/감정 (Raw Evidence)
2. 도출된 패턴 (Themes)
3. 가설 요소별 증거 매핑
4. 지지 vs 반증 데이터 요약
5. 가설 판단 결과 및 근거 (테이블로 구성)
6. 필요한 경우, 데이터에 기반한 수정 가설
   (새로운 주장 추가 금지)
